\chapter{Dalla sintassi alla grammatica}

In questo capitolo viene introdotta una sintassi concreta del linguaggio, progettata per essere leggibile e non ambigua, e successivamente formalizzata in EBNF. Infine, la specifica viene tradotta in una grammatica ANTLR, che costituisce la base per la generazione automatica del lexer e del parser utilizzati nel tool.

\section{Principi e requisiti della sintassi concreta}

La progettazione della sintassi concreta di Racing Choreographies è stata guidata dalle seguenti scelte progettuali secondo uno stile C-like:

\begin{itemize}
    \item \textbf{Separazione tra definizioni e coreografia principale}

    Un programma è composto da zero o più definizioni di procedura seguite da una coreografia principale identificata dalla keyword \texttt{main}. Questa scelta rende esplicito il punto di ingresso dell’esecuzione.

    \item \textbf{Uso di keyword esplicite}

    Costrutti che nella sintassi astratta sono espressi in forma vengono introdotti tramite keyword riservate: \texttt{proc}, \texttt{main}, \texttt{call}, \texttt{race}, \texttt{discharge}, \texttt{if}, \texttt{else}. Questa scelta facilita la distinzione tra identificatori e costrutti del linguaggio.

    \item \textbf{Struttura a blocchi}

    Le coreografie sono racchiuse tra parentesi graffe \texttt{\{...\}}.

    \item \textbf{La terminazione 0}

    Non viene rappresentata esplicitamente: la fine di un blocco corrisponde alla terminazione della coreografia contenuta.

    \item \textbf{Sequenzialità esplicita tramite punto e virgola}

    Le istruzioni e le chiamate di procedura sono terminate dal punto e virgola rendendo esplicita la sequenza delle azioni. I costrutti condizionali, invece, sono strutturati come statement a blocco e non richiedono il punto e virgola finale.

    \item \textbf{Distinzione tra condizionali locali e su race}

    Le due forme di condizionale previste dalla sintassi astratta vengono rese sintatticamente distinte:

\begin{verbatim}
if (p.e) { ... } else { ... }
if (s[k]) { ... } else { ... }
\end{verbatim}

    \item \textbf{Espressioni semplici}

    Le espressioni sono limitate a valori letterali o identificatori. Non sono presenti operatori aritmetici questo perché il linguaggio si concentra sulle interazioni tra processi e non sul calcolo.
\end{itemize}

\section{Sintassi concreta di Racing Choreographies}

Questa sezione mostra in modo sistematico la sintassi concreta del linguaggio Racing Choreographies.

\subsection*{Struttura del programma}

Un programma è composto da zero o più definizioni di procedura seguite da una coreografia principale identificata dalla keyword \texttt{main}.

Forma generale:

\begin{verbatim}
<procdef>*
main {
    <C>
}
\end{verbatim}

L’esecuzione del programma inizia sempre dal blocco associato a \texttt{main}.

\subsection*{Definizioni di procedura}

Una procedura è dichiarata tramite la keyword \texttt{proc}, seguita dal nome della procedura, dalla lista dei processi partecipanti e da un blocco che ne rappresenta il corpo.

Forma generale:

\begin{verbatim}
proc X(p1, p2, ..., pn) {
    <C>
}
\end{verbatim}

dove:

\begin{itemize}
    \item $X$ è il nome della procedura;
    \item $p_1, \dots, p_n$ sono i processi partecipanti;
    \item il blocco \texttt{\{ ... \}} contiene una coreografia.
\end{itemize}

\subsection*{Coreografie e blocchi}

Una coreografia è rappresentata da una sequenza di statement racchiusi tra parentesi graffe.

Forma generale:

\begin{verbatim}
{
    <stmt1>
    <stmt2>
    ...
}
\end{verbatim}

La terminazione della coreografia è implicita alla chiusura del blocco.

All’interno di un blocco è possibile scrivere:

\begin{itemize}
    \item istruzioni
    \item chiamate di procedura
    \item condizionali locali
    \item condizionali su esito di race
\end{itemize}

\subsection*{Chiamata di procedura}

La chiamata di procedura è resa esplicita dalla keyword \texttt{call}.

Forma generale:

\begin{verbatim}
call X(p1, p2, ..., pn);
\end{verbatim}

\subsection*{Condizionale locale}

Il condizionale locale valuta un’espressione appartenente a un processo specifico.

Forma generale:

\begin{verbatim}
if (p.e) {
    <C1>
} else {
    <C2>
}
\end{verbatim}

L’espressione \texttt{p.e} viene valutata localmente dal processo $p$.

\subsection*{Condizionale su esito di race}

Il condizionale su race dipende dall’esito di una race precedentemente eseguita.

Forma generale:

\begin{verbatim}
if (s[k]) {
    <C1>
} else {
    <C2>
}
\end{verbatim}

La condizione è vera se ha vinto il lato sinistro della race identificata da \texttt{s[k]} falsa altrimenti.

\subsection*{Istruzioni}

Le istruzioni sono le azioni elementari del linguaggio e terminano sempre con \texttt{;}.

\subsection*{Comunicazione}

Forma generale:

\begin{verbatim}
p.e -> q.x;
\end{verbatim}

Il processo $p$ valuta l’espressione $e$ e invia il valore risultante al processo $q$ che lo assegna alla variabile $x$.

\subsection*{Selezione}

Forma generale:

\begin{verbatim}
p -> q[L];
\end{verbatim}

Il processo $p$ invia la label $L$ al processo $q$.

\subsection*{Assegnamento}

Forma generale:

\begin{verbatim}
p.x = e;
\end{verbatim}

Il processo $p$ assegna alla propria variabile $x$ il valore dell’espressione $e$.

\subsection*{Race}

La race rappresenta una competizione tra due processi che offrono un valore a un processo ricevente.

Forma generale:

\begin{verbatim}
race s[k] : p.e , q.e' -> s.x;
\end{verbatim}

dove:

\begin{itemize}
    \item \texttt{s[k]} è l’identificatore della race;
    \item \texttt{p.e} e \texttt{q.e'} sono le offerte dei due processi competitori;
    \item \texttt{s.x} è la variabile in cui viene memorizzato il valore vincente.
\end{itemize}

\subsection*{Discharge}

Il discharge consente di ricevere il valore del processo perdente della race.

Forma generale:

\begin{verbatim}
discharge s[k] : p -> s.x;
\end{verbatim}

\subsection*{Espressioni}

Forma generale:

\[
\langle e \rangle ::= \langle value \rangle \mid \langle identifier \rangle
\]

I valori ammessi sono:

\begin{itemize}
    \item interi in base 10
    \item booleani
\end{itemize}

\section{Formalizzazione della sintassi in EBNF}

La sintassi concreta descritta nella sezione precedente viene ora formalizzata tramite una grammatica espressa in Extended Backus–Naur Form (EBNF).

L’uso della EBNF consente di specificare in modo formale e non ambiguo la struttura dei programmi e fornisce una descrizione formale completa della sintassi concreta del linguaggio.

\subsection*{Struttura del programma}

\[
\begin{array}{rcl}
Program &::=& \{ ProcDef \} \; Main \; ; \\
Main    &::=& \texttt{"main"} \; Block \; ;
\end{array}
\]

\subsection*{Definizioni di procedura}

\[
\begin{array}{rcl}
ProcDef    &::=& \texttt{"proc"} \; ProcName \; "(" \; ProcParams \; ")" \; Block \; ; \\
ProcParams &::=& Process \; \{ "," \; Process \} \; ;
\end{array}
\]

\subsection*{Blocchi e coreografie}

\[
Block ::= \texttt{"{"} \; \{ Stmt \} \; \texttt{"}"} \; ;
\]

\subsection*{Statement}

\[
\begin{array}{rcl}
Stmt &::=& InteractionStmt \\
     &\mid& CallStmt \\
     &\mid& IfLocalStmt \\
     &\mid& IfRaceStmt \; ;
\end{array}
\]

\subsection*{Condizionale locale}

\[
IfLocalStmt ::= \texttt{"if"} \; "(" \; ProcExpr \; ")" \; Block \; \texttt{"else"} \; Block \; ;
\]

\subsection*{Condizionale su esito di race}

\[
IfRaceStmt ::= \texttt{"if"} \; "(" \; RaceId \; ")" \; Block \; \texttt{"else"} \; Block \; ;
\]

\subsection*{Chiamata di procedura}

\[
\begin{array}{rcl}
CallStmt &::=& \texttt{"call"} \; ProcName \; "(" \; ProcArgs \; ")" \; ";" \; ; \\
ProcArgs &::=& Process \; \{ "," \; Process \} \; ;
\end{array}
\]

\subsection*{Istruzioni}

\[
\begin{array}{rcl}
InteractionStmt &::=& Interaction \; ";" \; ; \\
Interaction &::=& Comm \\
            &\mid& Select \\
            &\mid& Assign \\
            &\mid& Race \\
            &\mid& Discharge \; ;
\end{array}
\]

\subsection*{Comunicazione}

\[
Comm ::= ProcExpr \; \texttt{"->"} \; ProcVar \; ;
\]

\subsection*{Selezione}

\[
Select ::= Process \; \texttt{"->"} \; Process \; "[" \; Label \; "]" \; ;
\]

\subsection*{Assegnamento}

\[
Assign ::= ProcVar \; "=" \; Expr \; ;
\]

\subsection*{Race}

\[
Race ::= \texttt{"race"} \; RaceId \; ":" \; ProcExpr \; "," \; ProcExpr \; \texttt{"->"} \; ProcVar \; ;
\]

\subsection*{Discharge}

\[
Discharge ::= \texttt{"discharge"} \; RaceId \; ":" \; Process \; \texttt{"->"} \; ProcVar \; ;
\]

\subsection*{Espressioni}

\[
\begin{array}{rcl}
ProcExpr &::=& Process \; "." \; Expr \; ; \\
Expr     &::=& Value \\
         &\mid& Var \; ;
\end{array}
\]

\subsection*{Variabili localizzate}

\[
ProcVar ::= Process \; "." \; Var \; ;
\]

\subsection*{Identificatori}

\[
\begin{array}{rcl}
ProcName &::=& Identifier \; ; \\
Process  &::=& Identifier \; ; \\
Var      &::=& Identifier \; ; \\
Label    &::=& Identifier \; ;
\end{array}
\]

\subsection*{Identificatore di race}

\[
\begin{array}{rcl}
RaceId  &::=& Process \; "[" \; RaceKey \; "]" \; ; \\
RaceKey &::=& Identifier \; ;
\end{array}
\]

\subsection*{Lessico di base}

\[
\begin{array}{rcl}
Identifier &::=& Letter \; \{ Letter \mid Digit \mid "_" \} \; ; \\
Value      &::=& Integer \mid Boolean \; ; \\
Integer    &::=& Digit \; \{ Digit \} \; ; \\
Boolean    &::=& \texttt{"true"} \mid \texttt{"false"} \; ;
\end{array}
\]

\section{Traduzione della EBNF in grammatica ANTLR}

La grammatica EBNF presentata nella sezione precedente definisce in modo formale la struttura sintattica del linguaggio. Per poter generare un parser è necessario tradurre tale specifica in una grammatica compatibile con ANTLR.

ANTLR utilizza una notazione propria concettualmente molto vicina alla EBNF. La traduzione consiste principalmente in:

\begin{itemize}
    \item separazione tra regole sintattiche (parser rules) e regole lessicali (lexer rules);
    \item introduzione esplicita dei token per keyword e simboli;
    \item definizione delle regole per la gestione degli identificatori e dei valori letterali;
    \item eliminazione di ambiguità lessicali;
    \item la necessità di una regola di ingresso (\texttt{program}) che includa \texttt{EOF}.
\end{itemize}

\subsection*{Struttura generale}

\begin{verbatim}
program
  : procDef* mainDef EOF
  ;
\end{verbatim}

\[
Program ::= \{ ProcDef \} \; Main \; ;
\]

\begin{verbatim}
mainDef
  : MAIN block
  ;
\end{verbatim}

\subsection*{Definizione delle procedure}

\begin{verbatim}
procDef
  : PROC procName LPAREN procParams RPAREN block
  ;
\end{verbatim}

\begin{verbatim}
procParams
  : process (COMMA process)*
  ;
\end{verbatim}

La notazione \texttt{(...)*} implementa direttamente la ripetizione \{ ... \} della EBNF.

\subsection*{Blocchi e statement}

\begin{verbatim}
block
  : LBRACE stmt* RBRACE
  ;
\end{verbatim}

\begin{verbatim}
stmt
  : interactionStmt
  | callStmt
  | ifLocalStmt
  | ifRaceStmt
  ;
\end{verbatim}

\subsection*{Condizionali}

\begin{verbatim}
ifLocalStmt
  : IF LPAREN procExpr RPAREN block ELSE block
  ;
\end{verbatim}

\begin{verbatim}
ifRaceStmt
  : IF LPAREN raceId RPAREN block ELSE block
  ;
\end{verbatim}

\subsection*{Istruzioni}

\begin{verbatim}
interactionStmt
  : interaction SEMI
  ;
\end{verbatim}

\begin{verbatim}
interaction
  : comm
  | select
  | assign
  | race
  | discharge
  ;
\end{verbatim}

\subsection*{Comunicazione}

\begin{verbatim}
comm
  : procExpr ARROW procVar
  ;
\end{verbatim}

\subsection*{Race}

\begin{verbatim}
race
  : RACE raceId COLON procExpr COMMA procExpr ARROW procVar
  ;
\end{verbatim}

\subsection*{Discharge}

\begin{verbatim}
discharge
  : DISCHARGE raceId COLON process ARROW procVar
  ;
\end{verbatim}

\subsection*{Regole lessicali}

\begin{verbatim}
MAIN      : 'main' ;
PROC      : 'proc' ;
CALL      : 'call' ;
IF        : 'if' ;
ELSE      : 'else' ;
RACE      : 'race' ;
DISCHARGE : 'discharge' ;
TRUE      : 'true' ;
FALSE     : 'false' ;
ID        : [a-zA-Z] [a-zA-Z0-9_]* ;
\end{verbatim}

\subsection*{Gestione del lessico}

\begin{verbatim}
INT : [0-9]+ ;

WS            : [ \t\r\n]+ -> skip ;
LINE_COMMENT  : '//' ~[\r\n]* -> skip ;
BLOCK_COMMENT : '/*' .*? '*/' -> skip ;
\end{verbatim}

\section{Esempio di programma}

Per illustrare in modo unitario la sintassi concreta e la sua formalizzazione, si presenta un esempio completo di programma in Racing Choreographies.

Il programma descrive una semplice interazione tra quattro processi: un client $c$, due worker $w1$ e $w2$, e un server $s$.

Il client invia una richiesta al server. I due worker ricevono la richiesta e competono per fornire una risposta al server tramite una race. Il primo worker che consegna il proprio valore determina il risultato principale, che viene inoltrato al client. Il valore del worker perdente non viene viene recuperato tramite il costrutto di discharge e memorizzato separatamente.

\begin{verbatim}
proc Request(c, w1, w2, s) {

    c.req -> s.req;

    w1.r = req;
    w2.r = req;

    race s[k] : w1.r , w2.r -> s.ans;

    if (s[k]) {
        s.ans -> c.res;
        s -> c[FromW1];
        discharge s[k] : w2 -> s.lost;
    } else {
        s.ans -> c.res;
        s -> c[FromW2];
        discharge s[k] : w1 -> s.lost;
    }
}

main {
    call Request(c, w1, w2, s);
}
\end{verbatim}
